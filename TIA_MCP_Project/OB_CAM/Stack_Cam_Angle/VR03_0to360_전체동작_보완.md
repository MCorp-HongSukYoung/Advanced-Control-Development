## 0) 한 줄 결론

> **VR03 1회전(0°→360°) = 4-Phase 사이클 중 RUN Phase 1회.**  
> LC_RUN과 RA_RUN이 각각 VR03을 1회전시키며, P&P 12축 + Reverse 4축 + Stack Table Swing 4축이 VR03 슬레이브로 동기된다. Reverse R/Z는 GET(20°~180°)·PUT(180°~340°)·Dwell(나머지) 세 구간으로 나뉘어 동작한다.

---

## 1) 4-Phase 사이클 전체 구조

```
[LC_RUN] → [LC_STACK] → [RA_RUN] → [RA_STACK]  × N층
  VR03↑       VR04↑       VR03↑       VR04↑
```

| Phase | 마스터 | 주 동작 | Reverse 역할 |
|-------|--------|---------|-------------|
| **LC_RUN** | VR03 | Cathode PUT + Anode GET | Cathode Reverse PUT(180°~340°) / Anode Reverse GET(20°~180°) |
| **LC_STACK** | VR04 | Mandrel Cathode 전극 압착 | CAM 비활성 |
| **RA_RUN** | VR03 | Anode PUT + Cathode GET | Anode Reverse PUT(180°~340°) / Cathode Reverse GET(20°~180°) |
| **RA_STACK** | VR04 | Mandrel Anode 전극 수령·압착 | CAM 비활성 |

> VR01/VR02(Dsheet 검출)는 4개 Phase 내내 상시 회전.

```

                     │
                     │  MC_MOVERELATIVE(VR03, +360°)
                     ▼
     ┌──────────────────────────────────┐

     └──────────────────────────────────┘
                     │
    ┌────────────────┼──────────────────┐
    ▼                ▼                  ▼
[ Cathode P&P ]  [ Anode P&P ]    [ Stack Table ]
```

## 4) CAMIN 신호 구조

LC_RUN Phase에서 Reverse 축을 담당하는 CAMIN 신호는 **두 개**입니다:

| CAMIN 신호 | 사용 블록 | 적용 Reverse 축 | 동작 시점 |
|------------|-----------|----------------|----------|
| `STACK_LC_RUN_CAMIN` | `FB_Camin_STACK_LC_RUN` | Cathode Axis11/12 (PUT) + Anode Axis26/27 (GET) | 주 스태킹 LC_RUN Phase |
| `Single_LC_RUN_CAMIN` | `FB_Camin_Single_LC_RUN` | **Anode** Axis26/27 (PUT) | LC_Single 버퍼 채우기 Phase |

> **⚠️ Single_LC_RUN_CAMIN 주의**: 동일한 캠 프로파일(`Cam_STACK_LC_ReverseX_PUT`)이 Single Phase에서는 **Anode 축(Axis26/27)** 에 적용됩니다. STACK_LC_RUN의 Cathode Axis11/12와 대상 축이 다릅니다.

> **InSync 조건**: Reverse R/Z의 `CamIn.InSync`도 `STACK_LC/RA_RUN_InSync` 조건에 AND 포함 — Reverse 동기 실패 시 Phase 전환 차단.

## 6) VR03 각도별 동작 타임라인 (LC_RUN 기준, 교정판)

> **원본 파일과의 차이**: 비전 트리거 80°/170°가 LC↔RA 반전되어 있었으며, Align Vision은 230°가 아닌 120°~180° 구간입니다. `FB_Auto_Sub1_Stack_200` Step 230 코드로 확인.

| VR03 각도 | 비전 트리거 | LC 측 | RA 측 | Stack Table | Reverse |
|---|---|---|---|---|---|
| **0°** | — | PP1 매거진 GET 준비 / PP2 Stack PUT 준비 | PP1 Stack PUT 준비 / PP2 매거진 GET 준비 | Z 상승, 안착 자세 | Dwell (0°~20°) |
| **20°** | — | — | — | — | **Anode GET 시작**(Axis26/27) / **Cathode PUT 시작**(Axis12/11) |
| **0°~80°** | — | PP1 흡착 → Z 상승 → Reverse 방향 이동 / PP2 Stack 방향 이동 | PP1 Stack 방향 이동 / PP2 흡착 → Z 상승 → Reverse 방향 이동 | Z01/Z02 미세 승강 | Anode GET 추종 / Cathode PUT 추종 |

| **170°** | ✅ **LC** Vision<br>`DB_1st/2nd_Surface_Vision_LC.Trg[0]` | LC PP1 **Surface Vision** (Reverse Table) | — | — | — |

| **360° (=0°)** | — | `MC_MOVERELATIVE.Done` → sub-step 240 전환 | — | — | — |

### 비전 트리거 교정 근거

LC_RUN에서 **RA Vision이 먼저(80°), LC Vision이 나중(170°)** 트리거되는 이유:

```
LC_RUN (VR03 회전 중)            RA_RUN (다음 Phase)
  80°  → RA Vision 사전 검사  →  RA_RUN 시작 시 검사 데이터 준비 완료
  170° → LC Vision 검사        →  이번 LC 전극 품질 확인 완료
```

반대로 RA_RUN Phase에서는 **LC Vision이 80°, RA Vision이 170°** 에 트리거됩니다(`FB_Auto_Sub1_Stack_300` 확인).  
각 Phase가 **상대방 Vision을 먼저 쏘는 교차(Cross) 구조**로 다음 Phase 시작 전 검사 데이터를 미리 준비합니다.

## 8) GET/PUT 교차 구조 — LC_RUN 기준

### 한 사이클 안의 두 파이프라인

```
[ LC 전극 흐름 ]
매거진 ─▶ LC PP1(픽업) ─▶ Reverse Table(반전+Surface Vision) ─▶ AlignTable(직접 전달)
                                                                        │
                                                                  Align Vision
                                                                        │
                                                              LC PP2(픽업) ─▶ Stack Table ─▶ Gap Vision

[ RA 전극 흐름 ]
매거진 ─▶ RA PP2(픽업) ─▶ Reverse Table(반전+Surface Vision) ─▶ AlignTable(직접 전달)
                                                                        │
                                                                  Align Vision
                                                                        │
                                                              RA PP1(픽업) ─▶ Stack Table ─▶ Gap Vision
```

> 두 흐름은 **VR03 한 바퀴 동안 동시에 진행** (2-단 파이프라인).  
> **이번 사이클에 적층되는 전극 = 이전 사이클 Reverse 완료 전극**.  
> **이번 사이클 GET 전극 = 다음 사이클에 적층**.

> 두 파이프라인은 **VR03 한 바퀴 동안 동시에 진행**.  
> **이번 사이클에 적층되는 전극 = 이전 사이클 Reverse 완료 전극**.  
> **이번 사이클 GET 전극 = 다음 사이클에 적층**.

### GET/PUT 역할 매핑

| 사이드 | PP1 헤드 | PP2 헤드 | Reverse (LC_RUN) |
|---|---|---|---|
| **LC (Cathode)** | 매거진 픽업 → Reverse Table 전달 | AlignTable 픽업 → Stack Table 전달 | Axis12/11 **PUT** (180°~340°) → AlignTable 직접 전달 |
| **RA (Anode)** | AlignTable 픽업 → Stack Table 전달 | 매거진 픽업 → Reverse Table 전달 | Axis26/27 **GET** (20°~180°) → AlignTable 직접 전달 |

> RA_RUN에서는 Reverse 역할 반전: Axis26/27 PUT(180°~340°) / Axis12/11 GET(20°~180°)

## 9) 각 흐름 서술

### 9-1. LC (Cathode) — `PP1 → Reverse Table → AlignTable → PP2 → Stack Table`

1. **매거진 픽업**: `LC PP1`(Axis13 X / 14 Z) 매거진 위 Z 하강 → 흡착 → Z 상승.
2. **Reverse Table 전달**: LC PP1 → Reverse Table 위 Z 하강 → 전극 안착 (PP1 VAC OFF → Reverse VAC ON) → `ReverseR_PUT`(Axis12) R 축 회전으로 **반전**.
3. **Surface Vision**: 전극이 Reverse Table 위에 있는 동안 LC Surface Vision 검사 (VR03 >= 170°, `DB_1st/2nd_Surface_Vision_LC` 트리거 — 코드 확인).
4. **RA Vision 사전 검사**: VR03 >= 80°에서 RA Surface Vision 트리거 (`DB_1st/2nd_Surface_Vision_RA` — 코드 확인). 이 시점 RA 전극의 정확한 위치는 불명확하나, 파이프라인 구조상 Anode Reverse Table 위로 추정.
5. **AlignTable 직접 전달**: Reverse VAC OFF → AlignTable VAC ON — Reverse Table이 AlignTable로 **PP 없이 직접** 전극 전달 (Step 641).
6. **Align Vision**: AlignTable에서 Align Vision 검사 (`DB_Align_Vision`).
7. **Stack Table 전달**: `LC PP2`(Axis15 X / 16 Z) AlignTable에서 전극 픽업 → Stack Table 위로 이동 → 전극 안착.
8. **Gap Vision**: Stack Table에서 Gap Vision 검사.
9. **복귀**: PP1 → 다음 사이클 매거진 위, PP2 → 다음 사이클 Reverse 위.

### 9-2. RA (Anode) — `PP2 → Reverse Table → AlignTable → PP1 → Stack Table`

LC와 거울 대칭. **PP1/PP2의 GET/PUT 역할이 반대**임에 주의.

1. **매거진 픽업**: `RA PP2`(Axis30 X / 31 Z) 매거진 위 Z 하강 → 흡착 → Z 상승.
2. **Reverse Table 전달**: RA PP2 → Reverse Table 위 Z 하강 → 전극 안착 (PP2 VAC OFF → Reverse VAC ON) → `ReverseR_GET`(Axis27) R 축 회전으로 **반전**. (이름이 `_GET`: Reverse 출력이 PP1으로 전달되므로)
3. **Surface Vision**: 전극이 Reverse Table 위에 있는 동안 RA Surface Vision 검사 (VR03 >= 170°, `DB_1st/2nd_Surface_Vision_RA` 트리거 — 코드 확인).
4. **LC Vision 사전 검사**: VR03 >= 80°에서 LC Surface Vision 트리거 (`DB_1st/2nd_Surface_Vision_LC` — 코드 확인). 이 시점 LC 전극의 정확한 위치는 불명확하나, 파이프라인 구조상 Cathode Reverse Table 위로 추정.
5. **AlignTable 직접 전달**: Reverse VAC OFF → AlignTable VAC ON — Reverse Table이 AlignTable로 **PP 없이 직접** 전극 전달.
6. **Align Vision**: AlignTable에서 Align Vision 검사 (`DB_Align_Vision`).
7. **Stack Table 전달**: `RA PP1`(Axis28 X / 29 Z) AlignTable에서 전극 픽업 → Stack Table 위로 이동 → 전극 안착.
8. **Gap Vision**: Stack Table에서 Gap Vision 검사.
9. **복귀**: PP1 → 다음 사이클 Reverse 위, PP2 → 다음 사이클 매거진 위.

### 9-3. Stack Table Swing Roller — 분리막 덮기

1. **0°~180° · 대기/세팅**: Z 미세 승강으로 분리막 텐션 유지, R 다음 덮기 자세 정렬.
2. **120°~180° · 분리막 들어올림**: Z 상승 → 전극 안착 공간 확보.
3. **180°~320° · 덮기**: PP 헤드가 전극을 놓는 동안 R 회전·Z 하강 → 분리막을 전극 위로 덮음.
4. **320°~360° · 복귀**: Z 원위치, R 다음 사이클 시작 자세.
